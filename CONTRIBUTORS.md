## How to Contribute

Want to be listed here? We welcome contributions in many areas:

### 🧑‍💻 Code
- New features
- Bug fixes
- Performance improvements
- Test additions

See [CONTRIBUTING.md](CONTRIBUTING.md) for guidelines.

### 📚 Documentation
- README improvements
- Guide updates
- Example policies
- Translations

### 🐛 Testing
- Bug reports
- Feature testing
- Performance profiling
- Security review

### 💬 Community
- Answering questions in Discussions
- Writing blog posts
- Speaking at events
- Advocacy & sharing

## Recognition Tiers

We recognize contributions at multiple levels:

### 🌟 Tier 1: Single Contribution
- Merged PR
- Reported issue
- Documentation improvement
- Listed in CONTRIBUTORS.md

### 🔥 Tier 2: Regular Contributions
- 3+ merged PRs
- Ongoing community support
- Recognition in release notes
- GitHub contributor badge

### 💎 Tier 3: Core Contributor
- 10+ merged PRs or significant impact
- Leadership in specific areas
- Co-author on major features
- Consider for maintainer role

- 📢 Social media shout-out
- 🎁 Annual swag
- 🤝 Direct collaboration on roadmap

## Code of Conduct

All contributors agree to our [Code of Conduct](CODE_OF_CONDUCT.md).

We're committed to:
- Being respectful and inclusive
- Assuming good intent
- Focusing on ideas, not people
- Creating a safe space for everyone

Violations can be reported to dev@permissionpilot.app.

## Getting Started

New to contributing? Start here:

1. **Read** [CONTRIBUTING.md](CONTRIBUTING.md) for guidelines
2. **Look** for [good first issue](https://github.com/ChaitanyaJoshi1769/PermissionPilot/labels/good%20first%20issue) labels
3. **Join** [Discussions](https://github.com/ChaitanyaJoshi1769/PermissionPilot/discussions) to ask questions
4. **Fork** the repository
5. **Submit** a PR with your changes
